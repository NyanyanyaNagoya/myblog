PGV部落格工作流

July 3, 2026

簡單說明一下現在使用的工作流，包含三個階段：Publii ➔ GitHub ➔ Vercel。

概念說明

這個三階段工作流的核心邏輯是將「內容創作」、「檔案儲存」、「網站發布」三個環節完美分離，各自分工：

  • Publii是內容創作平台。Publii是開源的桌面端網站生成器，介面類似WordPress。可在本機端寫文章、套用主題、調整版面，過程完全視覺化，通常不需要程式碼，當然如果會寫CSS，可以把網站改得更好看一點。完成後，Publii 會將內容打包成體積小、無資料庫的靜態檔案。

  • GitHub扮演雲端儲存與中繼站的角色。GitHub 是程式碼儲存平台，Publii 會將打包好的靜態網站檔案，自動同步推送到 GitHub 上的儲存庫（Repository）中妥善保存，這也等於一種雲端備份。Github的專長是儲存資料，但是它不擅長快速發布網頁，所以這項工作改由Vercel擔任。

  • Vercel負責發布工作。Vercel是為靜態網站設計的頂級雲端部署平台。把Vercel 設定與GitHub 連動，只要偵測到 GitHub 上的檔案有更新，就會在幾秒鐘內自動將最新的網站內容發布到全球邊緣網路，讓讀者能極速造訪。

操作說明

初次建立網站時需要稍微設定，只要設定好，後續發布文章可以一鍵完成。

Step 1 - 設定Github:註冊並登入 GitHub，建立一個新的空白Repository。前往 GitHub 的開發者設定（Developer Settings），生成一組Personal Access Token (classic)，勾選 repo 的完整權限，將密碼記下。

Step 2 - 設定 Publii： 到 Publii 官網下載並安裝軟體。開啟軟體，建立新網站，寫一篇測試用文章。接著進入左側選單的Server設定：

  1. 伺服器類型（Server type）選擇GitHub Pages。

  2. 填入 GitHub Repository名稱以及分支（通常為 main 或 master）。

  3. 在 Token 欄位貼上剛剛生成的 Personal Access Token。

  4. 點選Save & Test Connection，確認連線成功後，按下左下角的Sync your website，將檔案推送到 GitHub。

Step 3 - 串接Vercel： 註冊 Vercel，可直接選擇使用 GitHub 帳號授權登入。

  1. 在 Vercel 後台點選Add New Project。

  2. 找到剛剛存放 Publii 檔案的 GitHub Repository，點選Import。

  3. 部署設定保持預設，點入Deploy。

  4. 稍候一下，Vercel會生成一個免費的 .vercel.app 網址（後續可自訂），部落格就正式上線了！

Step 4 專心寫作：設定完成後，未來只需要打開電腦裡的 Publii 專心寫作。寫完後點選Preview確認之後，就可點選Sync your website，新文章會推送到 GitHub，Vercel 會在背景默默更新網站。通常不到一分鐘就會更新完畢，去喝一杯茶剛剛好。

結語

這套工作流最大的優勢在於高效率、安全、便宜。因為是靜態網站，沒有資料庫漏洞可以鑽，安全性極高。家中電腦永遠有一份原始資料，不擔心大公司忽然關閉部落格服務，也是令人滿意的理由。其次，Vercel 提供的企業級CDN服務，確保無論在世界哪個角落開啟網站，速度都非常快。個人部落格流量不可能太高，幾乎免費。基於以上理由，這套工作流應該滿值得推廣的。
